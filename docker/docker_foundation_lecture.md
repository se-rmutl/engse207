## 2.2 Core Concepts (แนวคิดหลัก)

### 2.2.1 Image vs Container

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    IMAGE vs CONTAINER                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  🖼️ IMAGE (Read-only Template)        📦 CONTAINER (Running Instance)    │
│  ─────────────────────────────        ──────────────────────────────     │
│                                                                          │
│  • เหมือน "แม่พิมพ์" หรือ "Blueprint"    • เหมือน "สิ่งที่สร้างจากแม่พิมพ์"      │
│  • ไม่เปลี่ยนแปลง (Immutable)           • รันได้, หยุดได้, ลบได้              │
│  • ประกอบด้วย Layers                  • มี Writable Layer ด้านบน          │
│  • Download ได้จาก Registry           • สร้างจาก Image                   │
│                                                                          │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │                        ANALOGY                                    │   │
│  ├──────────────────────────────────────────────────────────────────┤   │
│  │                                                                   │   │
│  │     Image = Class (พิมพ์เขียว)         Container = Object (instance) │   │
│  │                                                                   │   │

│  │       engine: V8;                       myCar.start();            │   │

│  │                                                                   │   │
│  │     Image: node:20-alpine              Container: taskboard-api   │   │
│  │     (Template สำเร็จรูป)                  (App ที่กำลังรันอยู่)           │   │
│  │                                                                   │   │
│  └──────────────────────────────────────────────────────────────────┘   │
│                                                                          │
│  ┌──────────────────────────────────────────────────────────────────┐   │
│  │                     ONE IMAGE → MANY CONTAINERS                   │   │
│  │                                                                   │   │
│  │              ┌──────────────┐                                     │   │
│  │              │    IMAGE     │                                     │   │
│  │              │ node:20-alpine│                                    │   │
│  │              └───────┬──────┘                                     │   │
│  │                      │                                            │   │

│  │        │             │             │                              │   │
│  │        ▼             ▼             ▼                              │   │
│  │   ┌─────────┐   ┌─────────┐   ┌─────────┐                        │   │
│  │   │Container│   │Container│   │Container│                        │   │
│  │   │   #1    │   │   #2    │   │   #3    │                        │   │
```

### 2.2.2 Image Layers

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         IMAGE LAYERS                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  Docker Image ประกอบด้วยหลาย Layers ซ้อนกัน:                              │
│                                                                          │
│  ┌─────────────────────────────────────────────┐ ◄── Layer 5: App Code  │
│  │  COPY . /app                                │     (Your code)        │
│  │  CMD ["node", "server.js"]                  │     ~1 MB              │
│  ├─────────────────────────────────────────────┤                        │
│  │  RUN npm install                            │ ◄── Layer 4: Dependencies│
│  │  (node_modules)                             │     ~100 MB            │
│  ├─────────────────────────────────────────────┤                        │
│  │  WORKDIR /app                               │ ◄── Layer 3: Working Dir│
│  │  COPY package*.json ./                      │     ~1 KB              │
│  ├─────────────────────────────────────────────┤                        │
│  │  Node.js 20 + npm                           │ ◄── Layer 2: Runtime   │
│  │                                             │     ~150 MB            │
│  ├─────────────────────────────────────────────┤                        │
│  │  Alpine Linux (minimal OS)                  │ ◄── Layer 1: Base OS   │
│  │                                             │     ~5 MB              │
│  └─────────────────────────────────────────────┘                        │
│                                                                          │
│  ✅ ข้อดีของ Layer System:                                               │
│                                                                          │
│  1. 🔄 CACHING - ถ้า Layer ไม่เปลี่ยน → ใช้ Cache → Build เร็วขึ้น!        │
│                                                                          │
│  2. 💾 SHARING - หลาย Images ใช้ Layer เดียวกันได้                        │
│     ┌─────────┐   ┌─────────┐   ┌─────────┐                            │
│     │ Image A │   │ Image B │   │ Image C │                            │
│     │  App A  │   │  App B  │   │  App C  │                            │
│     └────┬────┘   └────┬────┘   └────┬────┘                            │
│          │             │             │                                  │
│          └─────────────┼─────────────┘                                  │
│                        ▼                                                │
│               ┌─────────────────┐                                       │
│               │  node:20-alpine │ ← Download ครั้งเดียว!                │
│               │   (Base Image)   │                                      │
│               └─────────────────┘                                       │
│                                                                          │
│  3. 📏 SMALL SIZE - แต่ละ Layer เก็บเฉพาะส่วนที่เปลี่ยน                    │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 10.1 Dockerfile Best Practices

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    DOCKERFILE BEST PRACTICES                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  1️⃣ USE SPECIFIC IMAGE TAGS                                             │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  ❌ Bad:   FROM node:latest                                      │    │
│  │  ✅ Good:  FROM node:20-alpine                                   │    │
│  │                                                                  │    │
│  │  เหตุผล: latest เปลี่ยนได้ตลอด → ไม่ reproducible                  │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  2️⃣ USE ALPINE IMAGES WHEN POSSIBLE                                     │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  ❌ node:20         (~350 MB)                                    │    │
│  │  ✅ node:20-alpine  (~130 MB)                                    │    │
│  │                                                                  │    │
│  │  Alpine = Minimal Linux distro → Smaller, more secure           │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  3️⃣ OPTIMIZE LAYER CACHING                                              │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  ❌ Bad (cache miss every time):                                 │    │
│  │     COPY . .                                                     │    │
│  │     RUN npm install                                              │    │
│  │                                                                  │    │
│  │  ✅ Good (cache hit if package.json unchanged):                  │    │
│  │     COPY package*.json ./                                        │    │
│  │     RUN npm install                                              │    │
│  │     COPY . .                                                     │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  4️⃣ RUN AS NON-ROOT USER                                                │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  # Create user                                                   │    │
│  │  RUN addgroup -S appgroup && adduser -S appuser -G appgroup      │    │
│  │                                                                  │    │
│  │  # Switch to user                                                │    │
│  │  USER appuser                                                    │    │
│  │                                                                  │    │
│  │  เหตุผล: Security - ถ้า container ถูก hack จะไม่ได้ root access  │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  5️⃣ USE .dockerignore                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  # .dockerignore                                                 │    │
│  │  node_modules                                                    │    │
│  │  .git                                                            │    │
│  │  .env                                                            │    │
│  │  *.md                                                            │    │
│  │  .gitignore                                                      │    │
│  │                                                                  │    │
│  │  เหตุผล: ลด build context size → Build เร็วขึ้น                   │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  6️⃣ COMBINE RUN COMMANDS                                                │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  ❌ Bad (3 layers):                                              │    │
│  │     RUN apt-get update                                           │    │
│  │     RUN apt-get install -y curl                                  │    │
│  │     RUN apt-get clean                                            │    │
│  │                                                                  │    │
│  │  ✅ Good (1 layer):                                              │    │
│  │     RUN apt-get update && \                                      │    │
│  │         apt-get install -y curl && \                             │    │
│  │         apt-get clean && \                                       │    │
│  │         rm -rf /var/lib/apt/lists/*                              │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  7️⃣ DON'T STORE SECRETS IN IMAGE                                        │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  ❌ Bad:  ENV DATABASE_PASSWORD=secret123                        │    │
│  │  ❌ Bad:  COPY .env .                                            │    │
│  │                                                                  │    │
│  │  ✅ Good: Pass at runtime                                        │    │
│  │     docker run -e DATABASE_PASSWORD=secret123 ...                │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

## 10.2 Security Best Practices

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    DOCKER SECURITY BEST PRACTICES                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  🔒 1. Use Official Images                                               │
│     • Always prefer official images from Docker Hub                     │
│     • Check for "Official Image" badge                                  │
│     • Verify publisher is trusted                                       │
│                                                                          │
│  🔒 2. Keep Images Updated                                               │
│     • Regularly rebuild with latest base images                         │
│     • Monitor for security vulnerabilities                              │
│     • Use `docker scout` or Snyk to scan images                         │
│                                                                          │
│  🔒 3. Minimize Attack Surface                                           │
│     • Use minimal base images (alpine, distroless)                      │
│     • Remove unnecessary packages                                       │
│     • Don't install dev tools in production                             │
│                                                                          │
│  🔒 4. Run as Non-Root                                                   │
│     • Create and use non-root user                                      │
│     • Set proper file permissions                                       │
│                                                                          │
│  🔒 5. Don't Expose Unnecessary Ports                                    │
│     • Only expose ports that are needed                                 │
│     • Use internal networks for inter-service communication             │
│                                                                          │
│  🔒 6. Use Secrets Management                                            │
│     • Never hardcode secrets in images                                  │
│     • Use Docker secrets or environment variables                       │
│     • Consider HashiCorp Vault for production                           │
│                                                                          │
│  🔒 7. Enable Resource Limits                                            │
│     • Set memory limits: --memory=512m                                  │
│     • Set CPU limits: --cpus=0.5                                        │
│     • Prevent DoS attacks                                               │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```
